Description du projet

Vous allez créer un site web qui permettra de prendre un rendez-vous suivant un lieu sélectionné sur une carte interactive pour une réparation d'ordinateur.

Le projet est une page de type One Page. Il y a une partie administration pour gérer les rendez-vous.

Processus de l'application

        L'utilisateur sélectionne un centre pour la réparation de son ordinateur

        Quand il cliquera sur le centre, une pop-up s’afficher avec les informations

            Coordonnées

            Position géographique

            Horaire

            Un bouton "Nous Contacter"

        L'utilisateur cliquera sur le bouton, une pop-up affichera un formulaire avec les champs :

            Nom et prénom

            Coordonnées

            Commentaires

        Une fois le formulaire soumis, un email sera envoyé sur une adresse email.

        Une alerte s’affichera un message de succès si le mail est envoyé, ou une alerte si le message en cas de problème.


Fonctionnalités

        L'administrateur doit pouvoir se connecter au back-office.

        L'administrateur doit pouvoir via un formulaire créer un lieu de réparation.

        L'administrateur doit pouvoir modifier et supprimer un lieu de réparation.


    L'utilisateur pourra sélectionner le lieu du rendez-vous sur une carte interactive (OpenStreetMap).

    L'utilisateur devra remplir un formulaire qui lui permettra de décrire son problème.



Les +

    Envoyer un email à l'utilisateur pour confirmer la prise de son rendez-vous

    Dans l'interface Admin lister les prochains rendez-vous et le nombre de jours restants avant le rendez-vous.

    Dans l’interface admin, lister les rendez-vous par centre.

    Faire en sorte de ne plus avoir besoin de la latitude et longitude dans la base de données.

    Ajouter une relation de type OneToMany. Un centre a / peut avoir plusieurs rendez-vous.
